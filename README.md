# Той шақыруы — Wedding Invitation Site

Статический сайт-приглашение на GitHub Pages с формой RSVP, которая сохраняет ответы в Google Sheets.

---

## Структура файлов

```
wedding/
├── index.html          ← весь сайт (HTML + CSS + JS)
├── apps-script/
│   └── Code.gs         ← серверная часть (Google Apps Script)
└── README.md
```

---

## Шаг 1 — Заполните контент сайта

Откройте `index.html` и найдите объект `CONFIG` в начале блока `<script>`.
Замените плейсхолдеры реальными данными:

```js
const CONFIG = {
  groom:        'Бауыржан',     // имя жениха
  bride:        'Айгерім',      // имя невесты
  date:         '14 маусым 2025 жыл',
  venueName:    'Grand Hall',
  venueAddress: 'Алматы қ., Абай к-сі 10',
  venueMapUrl:  'https://maps.google.com/?q=...',
  dresscode:    'Ресми киім',
  deadline:     '1 маусым 2025',
  // ...
};
```

Всё остальное — разметка, стили, анимации — менять не нужно.

## Шаг 2 — Google Sheets + Apps Script

### 2.1 Создайте таблицу

1. Откройте [sheets.google.com](https://sheets.google.com) и создайте новую таблицу.
2. Назовите файл `Қонақтар`.
3. Переименуйте первый лист в `RSVP` (двойной клик по вкладке внизу).
4. В строку 1 добавьте заголовки столбцов:

   | A | B | C | D | E | F |
   |---|---|---|---|---|---|
   | Дата | Аты-жөні | Келеді | Қанша адам | Тағам | Тілек |

### 2.2 Создайте Apps Script

5. В меню таблицы: **Расширения → Apps Script**.
6. Удалите весь код в редакторе.
7. Скопируйте и вставьте содержимое файла `apps-script/Code.gs`.
8. Нажмите **Сохранить** (Ctrl+S).

### 2.3 Разверните веб-приложение

9. Нажмите **Развернуть → Новое развёртывание**.
10. Тип: **Веб-приложение**.
11. Настройки:
    - Выполнять как: **Я**
    - Доступ: **Все** (в том числе анонимные пользователи)
12. Нажмите **Развернуть**.
13. Выдайте разрешения (Google попросит авторизоваться).
14. **Скопируйте URL развёртывания** — он выглядит так:
    ```
    https://script.google.com/macros/s/AKfycby.../exec
    ```

### 2.4 Вставьте URL в сайт

15. Откройте `index.html`, найдите строку в начале `<script>`:
    ```js
    const APPS_SCRIPT_URL = '';
    ```
16. Вставьте скопированный URL внутрь кавычек:
    ```js
    const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/AKfycby.../exec';
    ```

### 2.5 Проверьте интеграцию

Откройте URL в браузере — должен вернуться JSON:
```json
{ "status": "ok", "message": "Wedding RSVP endpoint is running." }
```

Затем заполните форму на сайте и убедитесь, что строка появилась в таблице.

> **Важно:** при каждом изменении кода Apps Script нужно создавать **новое развёртывание**
> (Развернуть → Управление развёртываниями → Изменить → Новая версия),
> иначе старый URL продолжит использовать прежний код.

## Шаг 3 — GitHub Pages

### 3.1 Создайте репозиторий

1. Откройте [github.com/new](https://github.com/new).
2. Назовите репозиторий: `wedding` (или любое другое имя).
3. Установите видимость **Public** (для GitHub Pages бесплатного плана).
4. Нажмите **Create repository**.

### 3.2 Загрузите файлы

```bash
cd wedding
git init
git add index.html apps-script/Code.gs README.md
git commit -m "init: wedding invitation site"
git branch -M main
git remote add origin https://github.com/ВАШ_НИК/wedding.git
git push -u origin main
```

### 3.3 Включите GitHub Pages

5. Перейдите в репозиторий → **Settings → Pages**.
6. Source: **Deploy from a branch**.
7. Branch: `main` / `/ (root)`.
8. Нажмите **Save**.

Через 1–2 минуты сайт будет доступен по адресу:
```
https://ВАШ_НИК.github.io/wedding/
```

## Обновление сайта после публикации

Любые правки в `index.html` публикуются командой:

```bash
git add index.html
git commit -m "update content"
git push
```

GitHub Pages обновится автоматически в течение 1–2 минут.

## Частые вопросы

**Форма отправляет, но строка не появляется в таблице.**
Проверьте: имя листа точно `RSVP` (регистр важен). В Apps Script откройте
«Выполнения» — там будет лог ошибки.

**CORS ошибка в консоли браузера.**
Убедитесь, что в настройках развёртывания выбрано «Доступ: Все».
Пересоздайте развёртывание, если меняли настройки.

**Сайт показывает старую версию.**
GitHub Pages кешируется до 10 минут. Сделайте hard refresh (Ctrl+Shift+R).
